Systematic use of the same title This pattern is identified when multiple records are found within the dataset that have the same dc:title value (ignoring case). If a record has multiple titles, of which just one is duplicated in another record, this issue is still raised. Equal title and description fields This pattern is identified for a record when it has a dc:title value that is equal to one of its dc:description values. If a record has multiple titles, of which just one is duplicated as a description, this issue is still raised. Near-identical title and description fields This pattern is identified for a record when it has a dc:title value that is almost equal to one of its dc:description values. If a record has multiple titles, of which just one is similar to a description, this issue is still raised. This issue is not raised if the title and the description are in fact identical, as this is covered by pattern P2. Unrecognizable title This pattern is identified for a record if it has a dc:title value that does not seem to be human-readable. If there are more than 5 characters in the title that are not either alphanumeric or simple spaces, this issue is raised. This pattern is also identified if a title fully contains a dc:identifier value. Non-meaningful title This pattern is identified for a record if it has a dc:title value that does not seem to be meaningful. If there are less than 3 characters in the title, this issue is raised. Missing description fields This pattern is identified for a record if it does not have any dc:description values, or if all its descriptions are in fact empty. Very short description This pattern is identified for a record if it has a dc:description value that is deemed too short. If the description contains 50 characters or less, this issue is raised. Extremely long values This pattern is identified for a record if it has a dc:title value that is deemed too long. If the title contains more than 70 characters, this issue is raised.
